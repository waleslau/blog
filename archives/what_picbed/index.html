给无处安放的图片找个家

waleslau 2019-07-21 in 摸鱼日常 · Views

忙活了几天，把备案什么的都弄好后，发现一个问题，我这个博客是阿里云的学生机，1M宽带的小水管，加载网页的时候，图片如果太多的话，对博客的反应速度影响不小。为了减少我的服务器的负载、加速网站图片的加载，我就打起了图床的主意。下面就介绍一下我所知道的图床

开始吧

常见公共图床

各种图床站百度一搜一大把，我只列了一些我试用过的，也是比较有名的

微博图床

免费!没有容量限制，全网 CDN 加速，支持 HTTPS，到哪里都很快。但是免费的服务也有不足的地方，上传的图片会被转成 jpg，图片中可能加上了肉眼难以识别的水印，另外微博的图片鉴别服务也可能会随时删除你的图片。

  • 速度很快

  • 支持 HTTPS

  • 上传地址

  • Chrome插件

SM.MS

口碑挺好的

  • 速度不错

  • 支持HTTPS

  • 每个文件最多5 MB。每次最多同时上传10个文件。其他没啥限制

  • 上传地址

imgur

这是国外一家老牌的图片存储服务商，口碑不错，

  • 国外速度很快，但是国内速度很不稳定

  • 支持 HTTPS

  • 需要注册

  • 上传地址

聚合图床

这个图床本身不提供图床服务，就像它的名字一样，聚合图床，每次图片会同步上传到淘宝微博等十几个的图床，即便原先的图床挂了，也可以通过搜索找回当初其他图床上的文件，保证万无一失。

  • 上传链接

imgURL

  • 单张图片限制大小为5M，游客限制每天上传10张

  • 上传链接

GitHub

Github真的是一位全能型选手，可以托管静态网站，可以当云存储，还能做图床，(GitHub：？？？)

  • 国内还行，有时抽风，国外速度很快

  • 支持HTTPS

  • 上传方式是新建一个 Repo，然后在 Issue 中传图（直接将图片拖动到 issue 输入框即可），然后复制你的图片地址啦

Gitee

既然Github都写上了，也不能忘了国内的Gitee啊，

  • 速度不错

  • 支持HTTPS

  • 使用方法同Github

  • 图片仅支持 JPG、GIF、PNG 格式，并且文件小于 2MB

自建图床

自建图床方案有两种，一种是利用云服务商提供的存储服务来作为图床，通过 API 来管理图片，另一种是在自己的云服务器上安装图床程序(前提是你的云服务器的带宽要给力)

我使用图床最看重的是稳定性，imgur的稳定性很好，但不适合在国内用；微博图床很棒，但前段时间出过一次问题(现在好像又恢复了？)，SM.MS运行好多年了，很稳定，口碑也不错；但我主要用的还是用云服务商提供的存储服务（我用的是阿里云的OSS），那些不是很重要的图片才会使用比如微博图床啦、SM.MS等公共图床，虽然这些公共图床现在用着都很棒，但我不想吧自己的重要数据交给别人控制。

适合自己的才是最好的。如果你有其他更好的图床服务，欢迎留言给我，我会补充进来。
